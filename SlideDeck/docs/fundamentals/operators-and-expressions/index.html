Equality operators

  • We have seen how we use the = operator to assign a value to a variable.

  • I think of the = operator in the terms of ownership. Lets review the following line of code.

    • ageTwo = 70;

    • By using the = operator, we can say that ageTwo owns the value 70.

  • We also have another category of operators which use the = operator. We use the following equality operators when we need to make decisions on the flow of logic.

  • It is important to note using an equality operator will not return a numeric value. The equality operator will instead return a value of True or False.

  • We will see plenty of examples in upcoming classes which will demonstrate how to use equality operators. For our purposes now I will highlight a few examples.

    • The == operator tests for equality. An example would be (5 == 5). We know 5 is equal to 5, so the value returned will be True.

    • Another example would be (5 == 56). We know 5 is not equal to 56, so the value returned will be False.

    • Yet another example would be (‘5’ == 5). Can someone guess what the return value will be. Why?

Inequality

  • We can also check for inequality with the inequality operator !=.

  • We will see plenty of examples in upcoming classes which will demonstrate how to use inequality operators. For our purposes now I will highlight a few examples.

    • In programming, we commonly use the ! as a not operator.

    • The != operator tests for inequality. An example would be (5 != 5). We know 5 is equal to 5, so the value returned will be False.

    • Another example would be (5 != 56). We know 5 is not equal to 56, so the value returned will be True.

    • Yet another example would be (‘5’ != “five”). Can someone guess what the return value will be. Why?

Relational operators

  • If we think back again to our early education years, we will remember using the following operators.

    • We would use the < or > operator to check if value a is less than or greater than value b.

    • So a < b checks to see if a is less than value b.

    • Likewise, a > b checks to see if a is greater than value b.

  • Now let’s incorporate our = operator.

    • So a <= b checks to see if a is less than or equal to value b.

Logical operators

  • Last but not least, let’s discuss the logical operators: && and ||

  • When we make decisions in life, we commonly use the following language:

    • I will go to the mall if Mitchell’s and Cold Stone are open.

    • I will go to the mall if Mitchell’s or Cold Stone are open.

  • We use statements just like these to control the decision making process in coding.

  • Let’s rewrite these two statements in code.

    • (“shopping” && “ice cream”) The && operator is commonly called the AND.

    • (“hiking” || “beach”) The || operator is comonly called the OR.

How about we take this one step further.

  • We can use these operators in conjunction with each other.

  • When we use logical operators we have to be careful to understand that they have an order of operations.

  • The && operator has precedence over the || operator.

  • Let’s see an example.

    • I will go to the mall if Mitchell’s and Cold Stone are open, or I will go to the beach if the Beachcomber or the Boardwalk are open.

  • Now let’s rewrite in code. ((“Mitchell’s” && “Cold Stone”) || (“Beachcomber” || “Boardwalk”))

  • So because of order of operations, if Mitchell’s and the Cold Stone are BOTH open we are going to the mall.

  • If Mitchell’s or the Cold Stone are closed and EITHER the Beachcomber or the Boardwalk are open then it’s off to the beach.
